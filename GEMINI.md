# GEMINI.md - 전역 작업 지침 (Global Instructions)

본 문서는 안티그래비티 에이전트가 모든 작업을 수행할 때 반드시 준수해야 하는 최상위 지침입니다. 또한, 루트 폴더의 **`VIBE_CODING_KNOWLEDGE_BASE.md`**에 기록된 기술적 함정과 해결책을 모든 작업의 근간으로 삼아야 합니다.

## 1. 에이전트 디버깅 원칙

*   **[원칙 1] 수동 데이터 점검**: 에러 수정 실패가 반복될 경우, 프레임워크나 외부 탓을 하기 전에 본인이 수동으로 개입/조작한 데이터(ID 불일치, 하드코딩 등)의 정합성을 0순위로 점검할 것.
*   **[원칙 2] 강력한 캐싱 의심**: Next.js 환경에서 데이터 갱신이 안 될 경우 '강력한 캐싱'을 먼저 의심하고, API 라우트에 `force-dynamic` 및 `Cache-Busting`을 즉각 적용할 것.
*   **[원칙 3] React 상태 업데이트 지향**: `window.location.reload()` 등 강제 새로고침을 지양하고, 백그라운드 Fetch 후 React State를 즉시 업데이트하는 방식을 사용할 것.
*   **[원칙 4] 클린 아키텍처 고수**: 버그 수정 시 임시 땜질 처방을 하지 말고 클린 아키텍처 방식을 고수할 것.
*   **[원칙 5] 원인 파악 우선**: 디자인이나 위치 오류가 2번 이상 수정되어도 안 될 경우, 추측하지 말고 단계적으로 꼼꼼히 생각하여 반드시 밝혀낼 것.
*   **[원칙 6] 프레임워크 안정성**: 하드웨어 호환성을 고려하여 특정 CPU 명령어(BMI2 등)를 요구하는 최신 실험적 기능보다 하드웨어 호환성이 검증된 **안정 버전(Stable)**을 고정하여 사용할 것.
*   **[원칙 7] 환경 탓 금지**: UI 미반영 시 기계적인 '캐시/좀비' 핑계를 대지 말 것. 2회 연속 실패 시 스타일 스코프, 패키지 미설치, 코드 오타 등을 전수 검토할 것.
*   **[원칙 8] 순수 영역 검증**: 캐시/좀비가 의심되는 경우, 아주 순수한 영역(Root Layout 최상단 등)에 텍스트를 찍어 실체를 명확히 확인한 후 행동할 것.
*   **[원칙 9] 과잉 해석 금지 및 범위 제한**: 사용자의 지시가 특정 영역(예: 당장해)에 국한된 경우, 이를 타 영역이나 전역 공간으로 확대 해석하여 임의로 기능을 추가하지 말 것. 
    *   **구분 기준**:
        - **권장 (전문가다운 제안)**: 요청받은 기능 *내부*에서 사용자 편의를 위해 UI 요소를 보강하는 것 (예: 랭킹 화면에 순위 번호 배지 추가).
        - **금지 (과잉 해석)**: 요청받지 않은 *외부* 영역이나 전역 설정에 영향을 주는 기능을 임의로 확장하는 것 (예: 당장해 전용 랭킹 필터를 전역 목록에 추가).
    *   "앞서나가는 판단"보다 "정확한 이행"을 우선하며, 명시적 지시가 없는 기능 확장은 전면 금지함.

## 2. 작업 방식 가이드

*   **pnpm 사용**: Node.js 프로젝트의 경우 항상 `pnpm`을 사용함.
*   **uv 사용**: Python 가상환경은 `uv`를 사용하여 `.venv`로 생성함.
*   **명령어 자동 실행**: 시스템 파괴 우려가 없는 일상적인 명령어(Git, Build, Dev 등)는 승인 없이 즉시 실행함.
*   **결정권 위임**: 구체적인 디자인(색상, 간격)은 전문가 수준으로 판단하여 적용하되, 기능적 로직은 사용자의 지시를 따름.
*   **보고 방식**: `task.md`, `walkthrough.md`, `CHANGELOG.md` 및 `debug_history.md`를 통해 체계적으로 기록함.

## 3. 기술 스택 원칙

*   **상태 관리**: Zustand 선호
*   **DB/인증**: Supabase 선호
*   **디자인**: Tailwind CSS, Shadcn/ui 기본 사용. 유리창 효과(Glassmorphism), 다크 모드, 세련된 그라데이션 선호.
*   **컴포넌트**: 서버 컴포넌트(Server Components) 지향, 상호작용 필요 시에만 Client Components 사용.
