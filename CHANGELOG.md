# CHANGELOG

## [2026-04-07] v3.2.8 - DB 동기화 완전 정상화
- **근본 원인 해결**: Prisma 버전 다운그레이드(7→6.4.1) 후 `prisma db push` 미실행으로 Neon DB에 테이블이 없어 POST 500 에러 연속 발생. `prisma db push` 실행으로 즉시 해결.
- **Prisma 6.4.1 안정화** (v3.2.6): 검증된 다른 프로젝트(Stock Chart Puzzle) 패턴 참조하여 불안정한 Prisma 7+Neon 어댑터 조합을 6.4.1 표준 방식으로 전환. `prisma.config.ts`, `debug-db` 라우트 정리.
- **안전한 동기화 병합 전략** (v3.2.7): `syncFromDB` 시 DB가 비어있을 경우 로컬 데이터를 보존하는 로직 추가. `initialFetchDone`을 `useState`로 전환하여 DB 로드 완료 즉시 자동 업로드 트리거 보장.

#### 로컬 개발 환경 안정화 및 인증 복구 (v3.2.0)
- **로컬 DB 500 에러 해결**: 
  - 로컬 개발 시 Neon 서버리스 어댑터의 WebSocket 충돌을 방지하기 위해, 개발 환경(`NODE_ENV !== 'production'`)에서는 표준 TCP 연결(`PrismaClient`)을 사용하도록 분기 처리했습니다.
- **인증 시스템 정상화**:
  - `authOptions`의 콜백에서 무거운 DB 작업을 제거하여 로그인 무한 로딩 현상을 해결했습니다.
  - 유저 DB 동기화 로직을 최적화하여 로그인 흐름이 끊기지 않도록 보완했습니다.
- **레이아웃 안정화 (CLS 차단)**: 
  - 초기 로딩 시 메인 아이콘이 크게 튀는 현상을 방지하기 위해 `mounted` 상태 제어 및 인라인 스타일 강제 적용을 완료했습니다.
- **환경 변수 로딩 안정화**: 
  - `DATABASE_URL` 로드 타이밍 이슈를 해결하기 위해 `prisma.ts`에서 런타임에 직접 구동 방식으로 개선했습니다.

#### Neon DB & 구글 로그인 서버 측 통합 완료 (v3.1.6)
- **아키텍처 전환**: 기존 Google Drive 동기화 방식에서 Vercel(Serverless) + Neon(PostgreSQL) + NextAuth 기반의 서버 측 데이터 관리 체계로 100% 전환했습니다.
- **보안 패치 및 빌드 안정화**: 
  - Vercel 빌드 차단 원인이었던 Next.js 보안 취약점을 해결하기 위해 **v16.2.2**로 업데이트했습니다.
  - Prisma 7.x의 파괴적 변경사항을 반영하여 **Neon 전용 어댑터**(`@prisma/adapter-neon`)를 도입하고 빌드 시 `prisma generate`를 자동 실행하도록 설정했습니다.
- **실시간 동기화 시스템**:
  - `SyncManager`를 통해 로그인 시 자동으로 DB 데이터를 불러오고, 변경 사항을 2초 간격으로 백그라운드에서 자동 저장합니다.
  - API 라우트에 `force-dynamic` 설정을 적용하여 빌드 타임 오류를 방지하고 실시간성을 확보했습니다.
- **UI/UX 개선**: 
  - 헤더에 구글 로그인/로그아웃 버튼 및 사용자 프로필 이미지(또는 이니셜)를 추가했습니다.
  - 로컬 실행 시 편의를 위해 프로세스 강제 종료 기능이 포함된 `run_app.bat`을 최신화했습니다.
